# دليل النشر والتشغيل النهائي — منظومة كفيل (كفيل) 🚀

مرحباً بك في دليل النشر والتهيئة النهائي لمنظومة **كفيل (Kafeel)**. توفر هذه الوثيقة دليلاً متكاملاً باللغة العربية لشرح كيفية تشغيل ونشر المنظومة بالكامل في بيئة الإنتاج الفعلي (Production Environment)، وربط قاعدة البيانات السحابية بـ Supabase، ونشر الواجهة بـ Vercel، وتهيئة بوابات الرسائل القصيرة SMS والدفع الإلكتروني المحلي.

---

## 1. تهيئة ونشر قاعدة البيانات (Supabase & PostgreSQL) 🗄️

تعتمد منظومة كفيل بالكامل على **Supabase** كمنصة خلفية وقاعدة بيانات، وتستخدم ميزات متقدمة مثل Row Level Security (RLS) ومستودعات التخزين الآمنة والـ Triggers والـ Views الأمنية.

### أ. إنشاء المشروع وربطه
1. قم بإنشاء مشروع جديد على منصة [Supabase](https://supabase.com).
2. اختر المنطقة الجغرافية الأقرب لعملائك (مثلاً: `eu-west-1` لضمان سرعة زمن الاستجابة).
3. قم بتثبيت وتنزيل Supabase CLI على جهازك المحلي وقم بربطه عبر:
   ```bash
   npx supabase login
   npx supabase link --project-ref <your-project-id>
   ```

### ب. تشغيل الهجرة وتطبيق الهيكل (Migrations)
يحتوي مجلد `supabase/migrations/` على كافة ملفات الهجرة اللازمة بالترتيب الفني لتهيئة الجداول، وسياسات الأمان RLS، والدوال التلقائية.
قم بتطبيق ملفات الهجرة على سيرفر الإنتاج مباشرة عبر سطر الأوامر:
```bash
npx supabase db push
```

> [!IMPORTANT]
> في حال التهيئة اليدوية أو الرغبة في إعادة البناء بالكامل، يمكنك استخدام الملف `supabase/schema.sql` كمرجع نهائي وموحد وشامل لتشغيل الاستعلامات داخل **SQL Editor** في لوحة تحكم Supabase.

### ج. تهيئة مستودعات التخزين (Storage Buckets)
يجب تهيئة مستودعي تخزين أساسيين في لوحة Supabase مع تفعيل الوصول العام للقراءة وتأمين الرفع بـ RLS:
1. **`documents`**: لحفظ وتخزين مستندات الزبائن والضامنين المرفوعة (الملفات والبطاقات الشخصية).
2. **`settlement-checks`**: لحفظ وتخزين صور صكوك وإيصالات ضمان تسويات ما بعد البيع.

يمكنك تفعيل الهيكل البرمجي للمستودعات عبر تشغيل ملف الهجرة `09_setup_storage_buckets.sql` أو إنشائهما يدوياً من التبويب **Storage** وجعلهما **Public**.

## 2. نشر الواجهة الأمامية (Frontend Deployment) 🌐

تم بناء الواجهة باستخدام React (Vite) و TypeScript مع تصميم خفيف وسريع يعتمد بالكامل على CSS ونظام الثيمات الثنائي بدون أي حاجة لـ Tailwind.

### أ. المتغيرات البيئية المطلوبة (Environment Variables)
يجب تهيئة المتغيرات التالية في منصة الاستضافة (مثلاً Vercel أو Netlify):

| المتغير البيئي | الوصف | القيمة المثالية |
| :--- | :--- | :--- |
| `VITE_SUPABASE_URL` | رابط مشروع Supabase الخاص بك | `https://your-project-id.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | مفتاح Anon العام للاتصال الآمن | مفتاح API العام من تبويب API Settings |

### ب. النشر عبر Vercel (الموصى به)
1. قم بربط مستودع Git الخاص بالمشروع بحسابك في [Vercel](https://vercel.com).
2. اضبط إعدادات بناء المشروع كالتالي:
   - **Framework Preset**: `Vite`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
3. أضف المتغيرات البيئية المذكورة أعلاه في قسم **Environment Variables**.
4. اضغط على **Deploy** ليتم نشر الموقع وحفظه مع تفعيل شهادة SSL تلقائياً.

## 3. تهيئة وتكوين بوابات الـ SMS والرسائل 📡

تتم إدارة وتهيئة بوابات الرسائل التنبيهية بالكامل من خلال **لوحة تحكم مدير النظام العام (Master Admin Dashboard)** في تبويب **"البوابات والمزودين"**، حيث يتم حفظها محلياً وبشكل آمن لتوجيه الطلبات.

### أ. بوابة Twilio SMS
لتفعيل الإرسال عبر Twilio:
1. اختر **Twilio SMS Gateway** من قائمة المزودين.
2. في حقل **API Key**، أدخل بيانات الاعتماد بالتنسيق التالي: `AccountSid:AuthToken` (مثال: `ACxxxxxx:your_auth_token`).
3. في حقل **Sender ID**، أدخل رقم هاتف Twilio المخصص للإرسال الخاص بك أو معرف المرسل الأبجدي المعتمد.

### ب. بوابة BulkSMS
لتفعيل الإرسال عبر BulkSMS:
1. اختر **BulkSMS Gateway** من قائمة المزودين.
2. في حقل **API Key**، أدخل بيانات الاعتماد بالتنسيق التالي: `TokenId:TokenSecret` (مثال: `123456:your_token_secret`).
3. في حقل **Sender ID**، أدخل اسم المرسل المخصص لحسابك.

## 4. تهيئة بوابات الدفع الإلكتروني وتخصيص الباقات 💳

### أ. بوابات الدفع المحلية
تدعم المنظومة 3 خيارات دفع محلية معتمدة داخل ليبيا:
1. **سداد (SADAD)**
2. **تداول (Tadawul)**
3. **ادفع لي (Edfa3ly)**
لتهيئة البوابات، يقوم مسؤول النظام بإدخال **رقم التاجر التعريفي (Merchant ID)** في لوحة الإدارة العامة ليتم توجيه وتأكيد عمليات دفع الاشتراكات تلقائياً.

### ب. تخصيص باقات الـ SaaS وحجم المستخدمين
تأتي المنظومة بـ 3 مستويات باقات أساسية، ويمكن لمدير النظام تعديل الأسعار، وميزات الباقات، والحد الأقصى لعدد المستخدمين المسموح به لكل مكتب (`max_users`) عبر تبويب **"باقات SaaS"** في لوحة التحكم:
- **الباقة الأساسية (BASIC)**: تدعم حجب الميزات الحساسة والتقارير المالية المتقدمة عبر بوابات الإقفال الزجاجية (`PremiumLock`).
- **الباقة الممتازة (PREMIUM)**: تفتح كافة التقارير وسجل التسويات.
- **الباقة غير المحدودة (UNLIMITED)**: تدعم الحجم الكامل للموظفين وتسهيلات كود المعارض المفتوحة.

## 5. توصيات الأمن والتحصين السيبراني في الإنتاج 🔐

تم تدعيم "كفيل" بضوابط أمنية متكاملة مسجلة في مركز المراقبة الأمنية:
1. **أمن وحماية سعر الشراء**: لا تقم بمشاركة مفاتيح الإدارة أو توفير صلاحيات المدير والمحاسب لغير الموثوقين، حيث يقوم جدول `transactions_raw` والـ `security_barrier VIEW` بحجب قيم `purchase_cost` عن موظفي البيانات والشركاء بشكل كامل وقاطع.
2. **تجميد الحسابات المشبوهة (Account Freezing)**: يقوم النظام آلياً برصد محاولات التخمين الفاشلة. يمكنك متابعة السجلات الأمنية عبر تبويب **الأمن والتحصين** في لوحة المسؤول، واتخاذ قرارات فورية بتعطيل أو تجميد أي مستخدم خارج النطاق الفني المسموح به.
3. **سجل الأعطال الآمن (Crash Telemetry)**: في حال حدوث أي خطأ برمجي، يتولى مكون `SecurityErrorBoundary` كتم تفاصيل الخطأ البرمجية عن المستخدم لمنع استغلال الثغرات، وتسجيلها بالكامل في قاعدة البيانات تحت جدول `system_runtime_errors` لمراجعتها وحلها بأمان.
